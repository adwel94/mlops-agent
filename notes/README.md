## 작성 규칙

- **코드·규칙·구현 세부는 여기 없다** — 그건 깃 히스토리·스킬(`.claude/skills/…`)·
  README·`cloud/RUNPOD_OPS.md`·CLAUDE.md 몫이다. 노트엔 "그때 왜 그렇게 됐나"의 개념만.
- **한 노트 = 한 개념.** 서로 다른 개념을 한 파일에 섞지 않는다.
- **지금 상태만 긍정형으로.** 과거 경위·"~였다가 바뀜" 식 히스토리는 개념 서술에 필요한
  만큼만. 참조는 아래(딛고 선 밑바탕)로만.
- 운영 사건은 `클라우드운영일기.md`에 시간순 append.
