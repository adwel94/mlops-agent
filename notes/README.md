# notes/ — 작업 여정·히스토리 인덱스

이 디렉토리는 **우리가 작업해온 여정과 히스토리**를 담는다(README=프로젝트 설명,
CLAUDE.md=협업 규약과 다르게). 문서는 **한 축 = 한 파일** 원칙으로 분리한다 — 서로 다른
축(진단·구현·운영)을 한 파일에 섞지 않는다. durable 규칙은 CLAUDE.md/README로 승격하고,
여기엔 "왜·어떻게 그렇게 됐나"의 근거·경위를 남긴다.

| 문서 | 축 | 내용 | 상태 |
|---|---|---|---|
| [`model-perf-journey.md`](model-perf-journey.md) | 진단 여정 | v1 저성능(~20%)을 관찰→진단→검증→처방까지. 결론: 병목=reaching(얼린 눈+단일 캠), 처방=손목캠 | 완결 (eval 0.32→0.54) |
| [`wrist-camera-impl.md`](wrist-camera-impl.md) | 구현 | 손목캠(panda_wristcam)을 범용·하위호환으로 파이프라인에 추가한 설계·코드·검증 | 완결 |
| [`cloud-ops-log.md`](cloud-ops-log.md) | 운영 | 클라우드(RunPod/HF) 인프라 사건과 교훈 (누적 로그) | 진행형 (append) |

## 축 배정 규칙 (새 문서 추가 시)

- **진단/분석** (왜 성능이 이런가, 무엇이 병목인가) → 여정 문서. 연속된 하나의 이야기면 한 파일.
- **기능 구현** (무엇을 어떻게 붙였나) → effort당 `*-impl.md` 한 개. 서로 다른 기능은 합치지 않는다.
- **운영 사건** (인프라 마찰·장애·과금) → `cloud-ops-log.md` 에 시간순 append (공유 축이라 한 파일).
